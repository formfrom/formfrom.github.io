SpringBoot程序的打包与运行

Windows环境

程序打包

在IDEA中执行Maven的package指令

或者执行Maven命令行语句

mvn package

就能得到一个名称为项目名-版本号.jar的一个jar包

程序运行

打好包之后，进入到包所在的目录进入cmd界面，执行以下命令即可运行程序

java -jar 项目名-版本号.jar

注：jar支持命令行启动需要依赖Maven插件（一般创建SpringBoot项目自带）

<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>

Linux环境

  • 程序打包之后将打包完成后的jar文件放到Linux系统中/usr/local目录下（建议）
  • 前台运行
    java -jar XXX.jar  
#特点：当前ssh窗口被锁定，可按CTRL + C打断程序运行，或直接关闭窗口，程序退出

java -jar XXX.jar & 
#特点：当前ssh窗口不被锁定，但是当窗口关闭时，程序中止运行。
  • 后台运行
    nohup java -jar test.jar &

#nohup 意思是不挂断运行命令,当账户退出或终端关闭时,程序仍然运行
#当用 nohup 命令执行作业时，缺省情况下该作业的所有输出被重定向到nohup.out的文件中
#除非另外指定了输出文件。


nohup java -jar test.jar >temp.txt &

#这种方法会把日志文件输入到你指定的文件中，没有则会自动创建
#解释下 >temp.txt
#command >out.file   
#command >out.file是将command的输出重定向到out.file文件，即输出内容不打印到屏幕上，而是输出到out.file文件中。

命令行启动查询端口-Windows

# 查询端口
netstat -ano
# 查询指定端口
netstat -ano |findstr "端口号"
# 根据进程PID查询进程名称
tasklist |findstr "进程PID号"
# 根据PID杀死任务
taskkill /F /PID "进程PID号"
# 根据进程名称杀死任务
taskkill -f -t -im "进程名称"

命令行启动查询端口-Linux

ps
# 查看服务器的进程信息

ps -e 
# 表示列出全部的进程

ps -f
# 显示全部的列（显示全字段）详细信息

ps -ef | grep 进程名称
# 根据进程名称显示进程信息    注：执行该命令至少有一个进程(搜索该命令的进程，就是这条命令本身)

kill PID
# 请求目标进程退出

kill -KILL PID
# 根据PID号强制杀死进程

kill -9 PID
# 根据PID号强制杀死进程

配置高级

临时属性配置

SpringBoot提供了灵活的配置方式，如果项目中有个别属性需要重新配置，可以使用临时属性的方式快速修改某些配置。方法也特别简单，在命令行方式启动的时候添加上对应参数就可以了。每个属性使用–开始，属性与属性之间使用空格分隔。

java -jar springboot.jar --server.port=8888 --logging.level.root=debug

开发环境中使用临时属性

打开SpringBoot引导类的运行界面，使用Alt+R快捷键显示Program arguments输入框，其中Program arguments对应的位置就是添加临时属性的。

我们运行main方法的时候，如果想使用main方法的参数，也就是下面的args参数，就是在上面这个位置添加的参数。

public static void main(String[] args) {
}

原来是这样，通过这个args就可以获取到参数。再来看我们的引导类是如何书写的

public static void main(String[] args) {
    SpringApplication.run(Application.class,args);
}

这个args参数居然传递给了run方法，看来在Idea中配置的临时参数就是通过这个位置传递到我们的程序中的。言外之意，这里如果不用这个args是不是就断开了外部传递临时属性的入口呢？是这样的，我们可以使用下面的调用方式，这样外部临时属性就无法进入到SpringBoot程序中了。

public static void main(String[] args) {
    SpringApplication.run(Application.class);
}

配置文件分类

  • SpringBoot中4级配置文件（高>>>>低）
    • 1级：file：config/application.yml
    • 2级：file：application.yml
    • 3级：classpath：config/application.yml
    • 4级：classpath：application.yml
  • 作用：
    • 1级与2级留做系统打包后设置通用属性
    • 3级4级用于系统开发阶段设置通用属性

自定义配置文件

方式一：使用临时属性设置配置文件名，注意仅仅是名称，不要带扩展名

方式二：使用临时属性设置配置文件路径，这个是全路径名

也可以设置加载多个配置文件

多环境开发

yaml

  • 单文件（使用—区分环境）
    # 应用环境
# 一般设置一些通用的配置
spring:
  profiles:
    active: pro

---
# 开发环境
spring:
  profiles: pro
server:
  port: 80
---
# 生产环境
spring:
  config:
    activate:
      on-profile: dev
server:
  port: 81
---
# 测试环境
spring:
  config:
    activate:
      on-profile: tes
server:
  port: 82
  • 多文件
    • application.yml（主配置文件）
      spring:
  profiles:
    active: pro
    • application-pro.yml（开发环境配置文件）
      server:
  port: 80
    • application-dev.yml（生产环境配置文件）
      server:
  port: 81
    • application-tes.yml（测试环境配置文件）
      server:
  port: 82

properties

  • 多文件
  • application.properties（主配置文件）
    spring.profiles.active=pro
  • application-pro.properties（开发环境配置文件）
    server.port=80
  • application-dev.properties（生产环境配置文件）
    server.port=81
  • application-tes.properties（测试环境配置文件）
    server.port=82

多环境开发独立配置文件书写技巧

  • 根据功能对配置文件中的信息进行拆分，并制作成独立的配置文件，命名规则如下
    • application.yml
    • application-dev.yml
    • application-devDB.yml
    • application-devMVC.yml
    • application-devRedis.yml
    • …….
  • 使用include属性在激活指定环境的情况下，同时对多个环境进行加载使其生效，多个环境间使用逗号分隔
    • 当主环境dev与其他环境有相同属性时，主环境属性生效；其他环境中有相同属性时，最后加载的环境属性生效
    spring:
    profiles:
        active: dev
        include: devDB,devMVC
  • 上面也有一个问题，切换环境比较麻烦。所以SpringBoot从2.4版开始使用group属性替代include属性，降低了配置书写量。
    spring:
  profiles:
    active: dev
    group:
      "dev": devDB,devMVC
      "pro": proDB,proMVC
      "tes": tesDB,tesMVC

多环境开发控制

  • 在maven中设置使用什么环境
    <profiles>
    <profile>
        <id>env_dev</id>
        <properties>
            <profile.active>dev</profile.active>
        </properties>
        <activation>
            <activeByDefault>true</activeByDefault>
        </activation>
    </profile>
    <profile>
        <id>env_pro</id>
        <properties>
            <profile.active>pro</profile.active>
        </properties>

    </profile>
    <profile>
        <id>env_tes</id>
        <properties>
            <profile.active>tes</profile.active>
        </properties>

    </profile>
</profiles>
  • 在Spring Boot中读取maven设置的环境即可（**@属性名@**就是读取maven中配置的属性值的语法格式）
    spring:
  profiles:
    active: @profile.active@
    group:
      "dev": devDB,devMVC
      "pro": proDB,proMVC
      "tes": tesDB,tesMVC

日志

在代码中使用日志工具记录日志

添加日志记录操作

package com.bbu.controller;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/books")
public class BookController {
    private static final Logger log = LoggerFactory.getLogger(BookController.class);
    @GetMapping
    public String helloController(){
        log.debug("debug");
        log.info("info");
        log.warn("warn");
        log.error("error");
        return "'model':'hello'";
    }
}

设置日志输出级别

Spring Boot默认日志级别为INFO。

日志的级别分为6种，分别是：

  • TRACE：运行堆栈信息，使用率低
  • DEBUG：程序员调试代码使用z
  • INFO：记录运维过程数据
  • WARN：记录运维过程报警数据
  • ERROR：记录错误堆栈信息
  • FATAL：灾难信息，合并计入ERROR

一般情况下，开发时候使用DEBUG，上线后使用INFO，运维信息记录使用WARN即可。下面就设置一下日志级别：

# 开启debug模式，输出调试信息，常用于检查系统运行情况（不推荐）
debug: true

# 设置日志级别，root表示根节点，即整体应用日志级别(推荐)
logging:
  level:
    root: debug

设置日志组

  • 控制指定包对应的日志输出级别，也可以直接控制指定包对应的日志输出级别

    # 设置日志级别，root表示根节点，即整体应用日志级别
logging:
  # 设置分组
  group:
    term: com.bbu.controller,com.bbu.service,con.bbu.dao
  level:
    root: debug
    # 设置具体某个包的日志级别
    com.bbu.controller: debug
    # 设置分组，对每个组设置日志级别
    term: debug

优化日志对象创建代码

package com.bbu.controller;

import lombok.extern.slf4j.Slf4j;
//import org.slf4j.Logger;
//import org.slf4j.LoggerFactory;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@Slf4j  //使用lombok提供的注解替换创建日志对象的语句
@RestController
@RequestMapping("/books")
public class BookController {
   
    //private static final Logger log = LoggerFactory.getLogger(BookController.class);
   
    @GetMapping
    public String helloController(){
        log.debug("debug");
        log.info("info");
        log.warn("warn");
        log.error("error");
        return "'model':'hello'";
    }
}

日志输出格式控制

  • PID：进程PID，用于表明当前操作所处的进程，当多服务同时记录日志时，该值可用于协助程序员调试程序。
  • 所属类/接口名：当前显示信息为Spring Boot重写后的信息，名称过长时，简化包名书写为首字母，甚至直接删除。
# 设置日志输出格式
logging:
  pattern:
    # console: "%d - %m %n"
    console: "%d %clr(%p) --- [%16t] %clr(%-40.40c){cyan} : %m %n"

日志文件

  • 在配置文件中设置

    logging:
    file:
        name: server.log

  • 设置日志文件大小，超过大小创建新的日志文件

    logging:
  file:
    name: springboot2_11_log/logs/server.log # 在根目录下的springboot2_11_log文件夹中的logs文件夹中创建文件
  logback:
    rolling policy:
      max-file-size: 4KB  # 每个日志文件最大4KB
      file-name-pattern: springboot2_11_log/logs/server.%d{yyyy-MM-dd}.%i.log # 新的日志文件名称的模板
