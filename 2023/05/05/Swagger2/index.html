Swagger2

发表于2023-05-05|更新于2023-05-05
|阅读量:

注：jdk1.8+，才能使用swagger2

一、创建一个SpringBoot-web项目

二、在pom文件中导入依赖坐标

<!--https://mvnrepository.com/artifact/io.springfox/springfox-swagger-ui -->
<dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-swagger-ui</artifactId>
    <version>2.9.2</version>
</dependency>
<!-- https://mvnrepository.com/artifact/io.springfox/springfox-swagger2 -->
<dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-swagger2</artifactId>
    <version>2.9.2</version>
</dependency>

或

<dependency>
    <groupId>io.springfox</groupId>
    <artifactId>springfox-boot-starter</artifactId>
    <version>3.0.0</version>
</dependency>

注：springboot与swagger存在版本兼容问题，使用2.9.2版本swagger+2.7.11版本springboot时需要在yml文件中进行如下设置，想要使用3.0.0版本swagger+更高版本的springboot需要进行其他设置（省略）

spring:
  mvc:
    path match:
      matching-strategy: ant_path_matcher

三、创建SwaggerController

package com.bbu.controller;

import com.bbu.pojo.Book;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/swagger")
public class SwaggerController {

    @GetMapping("/get")
    public String save1(String name, Integer id){
        return "'swagger':'test!'"+name+Integer.toString(id);
    }

    @GetMapping("/get/11")
    public String save11(){
        return "'swagger':'test!'";
    }

    @GetMapping("/get/12")
    public String save12(){
        return "'swagger':'test!'";
    }

    @GetMapping("/get/13")
    public Book save13(){
        return new Book();
    }

    @PostMapping ("/post")
    public Book save2(@RequestBody Book book){
        return book;
    }

    @PutMapping("/put")
    public String save3(){
        return "'swagger':'test!'";
    }

}

 */

}

五、启动项目，在浏览器中访问http://localhost:8080/swagger-ui.html，就可以得到swagger默认界面

六、配置文档信息

1. Swagger实例Bean是Docket，所以通过配置Docket实例来配置Swaggger。

//配置Swagger的Docket的Bean实例
@Bean
public Docket docket(){
  return new Docket(DocumentationType.SWAGGER_2);
}

2.通过apiInfo()属性配置文档信息

//配置Swagger信息=====》apiInfo
public ApiInfo apiInfo(){
  //作者信息
  Contact contact = new Contact("FormFrom", "http://www.formfrom.cn", "Colorful_mm@163.com");
  return new ApiInfo(
          "FormFrom's Swagger recorder",
          "Swagger2 Api Documentation",
          "v1.0.0",
          "http://www.formfrom.cn",
          contact,
          "Apache 2.0",
          "http://www.apache.org/licenses/LICENSE-2.0",
          new ArrayList());
}

3.Docket实例关联上apiInfo()

//配置Swagger的Docket的Bean实例
@Bean
public Docket docket(){
  return new Docket(DocumentationType.SWAGGER_2)
        .apiInfo(apiInfo());
}

4.实现效果

七、配置扫描接口

1.构建Docket时通过select()方法配置怎么扫描接口。

//配置Swagger的Docket的Bean实例
@Bean
public Docket docket(Environment environment){
  return new Docket(DocumentationType.SWAGGER_2)
          .apiInfo(apiInfo())
          .select()
          .apis(RequestHandlerSelectors.any())     //设置扫描接口的方式                                         
          .paths(PathSelectors.any())            //设置过滤路径
          .build();
}
  • api()方法中RequestHandlerSelectors的参数
    • basePackage:扫描指定的包
    • any:全部扫描
    • none:全部不扫描
    • withMethodAnnotation:根据方法上的注解类型扫描
    • withClassAnnotation:根据类上的注解类型扫描
  • paths()方法中PathSelectors的参数
    • ant:符合antPattern格式的路径才被扫描
    • any:全部扫描
    • none:全部不扫描
    • regex:符合正则表达式pathRegex的路径才被扫描

八、配置Swagger开关

1.通过enable()方法配置是否启用swagger，默认为true，如果是false，swagger将不能在浏览器中访问

@Bean
public Docket docket(Environment environment){
    return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo())
            .enable(true)
            .select()
            .apis(RequestHandlerSelectors.any())     //设置扫描接口的方式：
            .paths(PathSelectors.any())           //过滤路径
            .build();
}

2.设置当处在开发环境与测试环境中启用Swagger，生产环境不启用Swagger

@Bean
public Docket docket(Environment environment){

//开发环境与测试环境中使用Swagger
    //设置需要使用Swagger的环境
    Profiles profiles = Profiles.of("dev", "test");
    //通过environment.acceptsProfiles(profiles)判断是否处在自己设定的环境当中
    boolean flag = environment.acceptsProfiles(profiles);

    return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo())
            .enable(flag)
            .select()
            .apis(RequestHandlerSelectors.any())     //设置扫描接口的方式：
            .paths(PathSelectors.any())            //过滤路径
            .build();
}

九、配置API分组

1.如果没有配置分组，默认时default，可以通过groupName()方法配置分组名称

@Bean
public Docket docket(Environment environment){

//开发环境与测试环境中使用Swagger
    //设置需要使用Swagger的环境
    Profiles profiles = Profiles.of("dev", "test");
    //通过environment.acceptsProfiles(profiles)判断是否处在自己设定的环境当中
    boolean flag = environment.acceptsProfiles(profiles);

    return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo())
            .groupName("FormFrom")
            .enable(flag)
            .select()
            .apis(RequestHandlerSelectors.any())     //设置扫描接口的方式：
            .paths(PathSelectors.any())            //过滤路径
            .build();
}

2.设置多个分组（配置多个Docket即可）

//需要设置多个分组时，定义多个Docket实例即可
@Bean
public Docket docket1(){
    return new Docket(DocumentationType.SWAGGER_2).groupName("A");
}
@Bean
public Docket docket2(){
    return new Docket(DocumentationType.SWAGGER_2).groupName("B");
}
@Bean
public Docket docket3(){
    return new Docket(DocumentationType.SWAGGER_2).groupName("C");
}

//配置Swagger的Docket的Bean实例
@Bean
public Docket docket(Environment environment){

//开发环境与测试环境中使用Swagger
    //设置需要使用Swagger的环境
    Profiles profiles = Profiles.of("dev", "test");
    //通过environment.acceptsProfiles(profiles)判断是否处在自己设定的环境当中
    boolean flag = environment.acceptsProfiles(profiles);

    return new Docket(DocumentationType.SWAGGER_2)
            .apiInfo(apiInfo())
            .groupName("FormFrom")
            .enable(flag)
            .select()
            .apis(RequestHandlerSelectors.any())     //设置扫描接口的方式：
            .paths(PathSelectors.any())            //过滤路径
            .build();
}

十、实体配置（Models）

1.新建一个实体类

package com.bbu.pojo;

public class Book {
    private String bookName;
    private Integer price;

    @Override
    public String toString() {
        return "Book{" +
                "bookName='" + bookName + '\'' +
                ", price=" + price +
                '}';
    }

    public String getBookName() {
        return bookName;
    }

    public void setBookName(String bookName) {
        this.bookName = bookName;
    }

    public Integer getPrice() {
        return price;
    }

    public void setPrice(Integer price) {
        this.price = price;
    }
}

2.只要这个实体在请求接口的返回值上（即使是泛型），都能映射到实体项中：

@GetMapping("/get/13")
public Book save13(){
    return new Book();
}

十一、常用注解

  • @Api：用在请求的类上，表示对类的说明
    • tags = 说明该类的作用
    • value = 没意义，无需配置
  • @ApiOperation：用在请求的方法上，说明方法的用途作用
    • value = 说明方法的用途作用
    • notes = 方法的备注说明
  • @ApiParam：作用在参数、方法和字段上；表示对参数的添加元数据（说明或是否必填等）
    • name = 参数名
    • value = 参数说明
    • required = 是否必填
  • @ApiModel：用在响应实体类上，说明响应实体类的作用
  • @ApiModelProperty：用在属性上，描述响应实体类的属性
